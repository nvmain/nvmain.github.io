Decoders

The job of the decoder is to take a memory address and route the request to the correct memory location. The most simplistic decoder will simply look at bits in various positions and route requests accordingly. The figure below shows an example of this simplistic decoder.

The order of decoding is "C:H:K:B:R". Here, C = column, H = channel, K = rank, B = bank, R = row. This means that the column is chosen using the least significant bits, followed by the channel, rank, bank, and row in the most significant bits.

Different memory system designs may require more advanced decoders. For example, a memory system testing a giga-scale cache may forego the column field, instead opting to provide the set where the cached memory address resides, and a tag to verify the match. DNUCA memory systems may have decoders that keep track of where memory pages are and decode using a table rather than bit manipulation or arithmetic.
