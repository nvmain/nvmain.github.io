Contributing

If you would like to contribute some of your own code to NVMain, you can do so by creating a pull request.

Pull Requests

In order to create a pull request, you will need to create a fork of NVMain under bitbucket with your account. You can do this by going to the NVMain bitbucket page at http://bitbucket.org/mrp5060/nvmain and clicking the "Fork" button. All you need to do is give this fork a name, and it will be created automatically. For example, you can simply call this fork "nvmain_fork".

Next, you need to clone the forked repository as such:

hg clone https://myusername@bitbucket.org/myusername/nvmain_fork

You can commit changes as you make them using normal commands such as hg commit and hg push. The NVMain developers are NOT able to see your changes.

You can create a pull request on the bitbucket website by going to the NVMain fork on your personal bitbucket page and clicking the "Pull Requests" button in the upper right. Merge your branch with the repository "mrp5060/nvmain" using the branch "default".

Note that you may want to keep contributed changes in a separate branch from your own personal work, since the entire branch will be added to the NVMain master repository if your pull request is merged.

Pull requests merged into the repository should have your name on the commit log. Please keep any pull requests as clean as possible, following the existing coding style of the source code. Any pull requests with extraneous code or not following the style may be rejected. In some cases, we may pull features and clean up the code as a separate administrative commit.

And of course, contributions are much appreciated!
