Working With NV Main

NVMain is a live project with frequent changes to the code (i.e., bug fixes, code optimization, additional example modules, etc.). Therefore, this page has suggestions for working with NVMain to allow you to keep your code changes without preventing you from updating to the latest version.

NVMain's version control system uses Mercurial. These recommendations make use of mercurial's queues to keep track of your local changes and additions to the code.

To get started, you'll need to initialize queues. This step only needs to be done once, ever:

hg qinit

Next you create a queue or patch. This queue can be the name of your patch, project, paper, etc. With this patch queue, you can even create multiple patches for each project you are working on. The command below will initialize a patch named "dac.patch"

hg qnew dac.patch

As you make changes to the code, you can save them to the patch by refreshing the queue:

hg qrefresh

When you want to update NVMain or switch patches, you simply remove the patch using the qpop command, and update or push another patch using qpush. To update, you need to pull the latest version from the NVMain repository and update your local copy. Below is an example of updating:

hg qrefresh # Save your changes
hg qpop -a # Remove all patches with -a
hg pull # pull changes from NVMain repo
hg update # update local copy
hg qpush -a # Reapply my patches

Using queues will help keep NVMain up-to-date. Of course, these steps are just recommendations. If you do not plan on updating NVMain mid-project, you do not need to follow these steps.
