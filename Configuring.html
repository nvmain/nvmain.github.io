Configuring

NVMain uses a configuration script to determine run behavior. Most of the parameters are necessary to run, while some NVMObjects will have custom parameters (e.g., DRAM controllers have additional parameters), this wiki entry defines the most common parameters.

Memory Parameters

The memory parameters as used to define the memory architecture, for example, the number of banks, ranks, channels, the memory controller to use, and interconnection network.

  • BANKS is the number of banks in each memory device.
  • BPC is number of bits from each device on each clock cycle.
  • BusWidth is the number of data pins in a rank of memory.
  • CHANNELS is the number of channels in the memory subsystem.
  • CLK is the frequency of the interconnect to the memory.
  • COLS is the number of logical columns in each memory bank. Logical meaning selectable columns. Typically this will be the number of bitlines divided by the number of bits provided by each device in a rank.
  • CPUFreq is the frequency at which the memory controller will run. Typically this is the same as the CPU frequency you are simulating.
  • INTERCONNECT is the name of the interconnect to use.
  • UseRefresh is whether refresh is used in the memory subsystem.
  • MEM_CTL is the name of the memory controller to use.
  • MULT is the clock rate multiplier. The memory core will run at the frequency of MULT*CLK. This is defined by JEDEC as 1 for DDR, 2 for DDR2, and 4 for DDR3.
  • RANKS is the number of ranks on each channel.
  • RATE is the number of data outputs per clock cycle. Typically this is 2 for DDR and 1 for SDRAM.
  • RAW is the maximum number of activations that can occur in the time tRAW.
  • RBSize is the size of the row buffer in bytes. Note that RBSize / (device width * burst length) must be less than or equal to COLS.
  • RefreshRows is the number of rows to refresh for each refresh command.
  • ROWS is the number of wordlines in a each memory bank.
  • WriteMode is the method to use when writing to non-volatile a memories. The options are WriteThrough or WriteBack. WriteThrough will stall the memory bank during the write operation. WriteBack stalls the memory bank during the precharge operation. The stall time is tWP in both cases.

Timing Parameters

Timing parameters are in unit of memory clock cycles, tCK.

  • tAL is the time for posted commands. Typical this is either 0 or 1, depending if posted commands are supported.
  • tBURST is the time to burst a cacheline on the interconnect. This time depends on the disparity between the cacheline size and interconnect width.
  • tCAS is the time for data to be sensed by the sense amplifiers and output to the interconnect.
  • tCCD is the delay of the column decoder (a multiplexer) between the sense amplifiers and output circuitry.
  • tCMD is the time a command spends on the bus. Usually 1 memory clock cycle.
  • tCWD is the delay to switch the two-way input/output circuitry from output to input.
  • tOST is the time to switch the on-die termination. Applicable for DDRx memories, but not LPDDRx memories.
  • tRAS is the time needed to restore data read from memory cells back to memory cells in the case the data is lost. This is typically 0 for non-volatile memory, and around 35ns for DRAM.
  • tRAW is minimum amount of time for N activations to occur, where N is the value of the RAW parameter. For standard DRAM, this is the value for tFAW when RAW is set to 4.
  • tRC is not used in NVMain.
  • tRCD is the row activation time.
  • tREFW is the maximum amount of time between refreshes of the same row. Usually this is 64ms for DDRx DRAM, 32ms for LPDDRx DRAM. Non-volatile memory does not require refresh, and this parameter is ignored.
  • tRFC is the time for a refresh command to complete. Non-volatile memory does not require refresh, and this parameter is ignored.
  • tRP is the time to precharge the bitlines to the voltage required by the sense amplifiers.
  • tRRDR is the minimum time between activations when no write has occurred in the previous activation. Used to limit the peak current.
  • tRRDW is the minimum time between activations when a write occurred in the previous activation. Used to limit the peak current.
  • tRTP is the time between a read and precharge command. Needed to ensure the read burst completes and the sense amplifiers can now be used to equalize the bitlines. Must be at least tBURST.
  • tRTRS is a small delay to allow transmission line buses to discharge when switching between different bus drivers (i.e., different ranks). Normally 1 for transmission line interconnects.
  • tWP is the time for the write pulse to complete.
  • tWR is the time to recover from a write command before a precharge can be issued.
  • tWTR is the time between a write and a read command.

Additionally, timings related to low-power states may be needed.

  • tPD is the time to powerdown to any powerdown mode (precharge powerdown fast exit, precharge powerdown slow exit, active powerdown)
  • tXP is the powerup time from active powerdown or precharge powerdown fast exit
  • tXPDLL is the powerup time from precharge powerdown slow exit
  • tRDPDEN is the time from read to powerdown entry with a precharge issued (precharge powerdown)
  • tWRPDEN is the time from write to powerdown entry with no precharge issued (active powerdown)
  • tWPAPDEN is the time from write to powerdown entry with a precharge issued (precharge powerdown)

MLC Parameters

  • UniformWrites is used to enable variable write times (e.g., Program-and-Verify). Set this to false to enable.
  • MLCLevels is the number of bits in each cell.
  • ProgramMode specifies whether to use single-RESET-multiple-SET (SRMS) or single-SET-multiple-RESET (SSMR). Possible values are "SRMS" and "SSMR".
  • WPVariance is the variance on the number of SET pulses (for single-RESET-multiple-SET) or RESET pulses (for single-SET-multiple-RESET).
  • tWP0 is the RESET pulse time
  • tWP1 is the SET pulse time
  • nWP00 is the number of program pulses to write 00
  • nWP01 is the number of program pulses to write 01
  • nWP10 is the number of program pulses to write 10
  • nWP11 is the number of program pulses to write 11

Control Parameters

  • PrintGraphs will output ASCII timing diagrams for each bank when enabled.
  • PrintPreTrace will output a memory trace in NVMain format for memory requests previous to scheduling.
  • PreTraceFile specifies the path to the pretrace output file. If a relative path is used, it will be relative to the directory in which the configuration file is located.
  • EchoPreTrace will output the pretrace to stdout if set to true.
  • PeriodicStatsInterval will output NVMain stats every N memory cycles, where N is the value of this parameter.
  • TraceReader specifies the format of the trace file in the case of trace-based simulation.
  • EnduranceModel specifies the endurance model to use in the memory system. Unless you are testing endurance models, please use NullModel for the best simulation speed and memory usage.
  • EnduranceDist specifies the probability distribution to use when determining endurance for memory cells. Each distribution has different parameters. See Endurance.
  • InitPD will start memory banks in powered down mode if enabled.
