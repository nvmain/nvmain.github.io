NVM Object

The memory system design in NVMain is abstracted as a tree-like structure with the NVMain object at the root. The NVMain object is connected to Memory Controllers. Memory Controllers are connected to the interconnect. The interconnect can be connected to ranks, which are connected to devices, which are connected to banks. An abstract memory system is shown in the figure below.

This tree-like structure has each object being a parent for it's child objects. There is only one parent and multiple children. When memory requests are propogated through the simulator, incoming requests use a Decoder to select the child object. Generally this decoder is the address translator. Once requests are served, usually by banks at the endpoint, the response propogates up through the parent until the owner of the request is reached. When building a memory system with a generic NVMObject, the parent and children must be set.

NVMObject also has a generic interface to get statistics from each module. These statistics can include area, power, timing, etc.
